# Day 2 - Linux Terminal Practice

Today I worked on basic Linux file and directory management in the terminal. The goal was to get more comfortable creating folders and files, moving around the file system, and using commands without guessing.

## Tasks Completed
- Created a folder called `practice`
- Created 2 files inside the folder
- Renamed one file
- Copied one file
- Deleted one file
- Navigated into and out of folders confidently

## Lab Summary
I created a new directory using `mkdir`, changed into it with `cd`, and created files inside it with `touch`. I then used `mv` to rename a file, `cp` to copy a file, and `rm` to remove a file I no longer needed. I used `ls` throughout the lab to verify that my files and folders were changing as expected.

## Mistake and Fix
At first, I tried to create a folder using `touch`, which actually creates a file instead of a directory. Because of that, `cd makingfolder` failed. I corrected this by using `mkdir`, which is the proper command for creating directories.

## Commands Practiced
- `mkdir`
- `cd`
- `touch`
- `ls`
- `mv`
- `cp`
- `rm`

## What I Learned
This lab helped me better understand:
- the difference between files and directories
- when to use `touch` vs `mkdir`
- how to navigate the Linux file system more accurately
- how to verify my work using `ls`

## Reflection
This was a simple lab, but it helped build confidence with core Linux commands. Making and correcting a mistake also helped reinforce the purpose of each command, which made the learning more practical.
