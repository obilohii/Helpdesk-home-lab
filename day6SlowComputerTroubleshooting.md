# Day 6 — Task Manager, System Information, and Slow Computer Troubleshooting

## Goal
Today’s goal was to learn how to identify what may be causing a computer to feel slow by using built-in Windows tools like Task Manager, System Information, About Your PC, and Startup Apps.

## Tasks Completed
- Opened Task Manager
- Reviewed the main Task Manager tabs:
  - Processes
  - Performance
  - Startup
  - Users
  - Details
- Sorted running processes by:
  - CPU usage
  - Memory usage
  - Disk usage
- Identified which applications were using the most system resources
- Opened multiple apps at once and watched how Task Manager changed in real time
- Practiced ending a frozen or unnecessary app using **End Task**
- Found key computer specifications:
  - Total RAM
  - Processor
  - Storage size
  - Windows version
- Opened **System Information**
- Opened **About Your PC**
- Reviewed **Startup Apps** to see what launches when Windows starts

## Practice Scenarios

### Scenario 1: User says their laptop is slow
I practiced checking Task Manager to look for high CPU, memory, or disk usage. This helped me understand how to identify whether a slow computer may be caused by too many apps running, a resource-heavy program, or background processes.

### Scenario 2: An app is frozen
I practiced selecting a frozen or unnecessary application in Task Manager and using **End Task** to close it. This is a common help desk troubleshooting step when a program stops responding.

### Scenario 3: IT needs computer specs
I practiced finding and writing down the device specifications, including RAM, processor, storage, and Windows version. This is useful when documenting tickets or escalating issues.

## Mini Troubleshooting Checklist
When troubleshooting a slow computer, I can check:

1. Startup apps
2. CPU usage
3. Memory usage
4. Disk usage
5. Available storage
6. Restart the frozen app
7. Restart the computer if needed
8. Confirm whether performance improved

## What I Learned
Today I learned how Task Manager can be used to quickly understand what is happening on a computer. I also learned how to find system information that may be needed for support tickets, troubleshooting, or escalation.

## Takeaway
Task Manager is one of the most important tools for basic IT support. It helps identify slow performance, frozen apps, startup issues, and system resource usage.
