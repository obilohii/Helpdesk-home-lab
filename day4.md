# Day 4 - Windows System Tools and Basic Administration

Today I practiced using built-in Windows tools to better understand how a computer is set up, what is running in the background, and how system activity is tracked.

## Tasks Completed
- Found my system information by opening **Settings > System > About**
- Created or reviewed a local user account
- Opened **Task Manager** and identified running apps and background processes
- Opened **Services** and looked at running vs stopped services
- Opened **Event Viewer** and found a recent system log
- Wrote down what each tool is used for

## What I Practiced
For this lab, I focused on learning where important Windows system information is located and how to use a few core administrative tools. I started by opening the **About** section in Windows Settings to view system details like the device name, installed RAM, processor, system type, and Windows version.

I also reviewed a **local user account**, which helped me understand how user access is managed directly on a machine. After that, I opened **Task Manager** to see which apps were currently open and what background processes were running. This gave me a better understanding of how Windows shows system activity in real time.

Next, I opened **Services** and compared which services were running and which were stopped. This helped me see that many important Windows functions run in the background without direct user interaction. I then opened **Event Viewer** and found a recent system log to see how Windows records events, warnings, and other system activity.

## What Each Tool Is Used For
- **System > About**: Used to view basic information about the computer, such as the processor, RAM, device name, and Windows version.
- **Local User Accounts**: Used to manage who can sign in to the computer and what level of access they have.
- **Task Manager**: Used to identify apps you have open, running processes, and background activity on the computer.
- **Services**: Used to view and manage background services that help Windows and applications function, such as networking, printing, security, updates, and authentication.
- **Event Viewer**: Used to view logs of events that happen on the computer, including system events, warnings, and errors.

## What I Learned
I learned that Windows has several built-in tools that give you different levels of visibility into how a computer operates. **Task Manager** is useful for checking active applications and background processes, while **Services** shows the behind-the-scenes components that keep the operating system working. I also learned that **Event Viewer** is important for troubleshooting because it keeps a history of what has happened on the machine.

This lab helped me better understand the difference between what a user sees on the surface and what is happening in the background of the system.

## Why This Matters for IT and Cybersecurity
Learning these tools is important because they are commonly used in IT support, troubleshooting, and security work. Being able to check system information, review accounts, identify running processes, inspect services, and read logs builds a strong foundation for understanding how Windows systems behave and how to investigate issues when something goes wrong.
