## What I Learned

This lab helped me understand that Windows support is not just about knowing where things are located — it is about knowing which tool to use for the issue in front of you.

I learned that:
- **File Explorer** is essential for locating missing files, organizing folders, and recovering deleted items
- **Settings and Control Panel** both matter because users may need help in either location depending on the task
- **Startup Apps** can explain why a computer feels slow right after boot
- **Installed apps and default apps** are common areas for software troubleshooting
- **Local user accounts** are important for access control and account management
- **Administrator vs standard user permissions** directly affect what a user can and cannot do on a machine

I also learned that practicing these tasks hands-on makes Windows feel much less overwhelming. Repeating the steps helped me start thinking more like a support technician instead of a casual user.

## Notes from Today
- Downloads, Desktop, and Documents are some of the first places to check when files seem missing
- Search filters like name, type, and date modified can make finding files much easier
- Startup apps matter because too many apps loading at sign-in can slow the system down
- Settings and Control Panel overlap in some places, but both are still worth knowing
- Standard users are more limited than administrators, which is important for troubleshooting permission issues
- Writing down steps for account creation and password changes is good practice for support documentation
